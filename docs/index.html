Claude Code, in a box.

Run Claude Code at full permissions inside a locked-down Docker container. An iptables firewall blocks everything except the handful of domains it actually needs. Your code stays on the host. Your databases stay unreachable. The Schwartz stays strong.

The problem

Skip permissions is fast. It's also how you wake up to a dropped table.

--dangerously-skip-permissions makes Claude Code fly. But an agent running unattended on your laptop can reach your production database, read every .env you own, push to any remote, and phone home to anywhere. "It probably won't" is not a security model.

🔥

The naive way

YOLO mode on the host. Full network. Full filesystem. One confidently-wrong command away from a very bad afternoon.

🐢

The paranoid way

Approve every tool call by hand. Safe, but you've turned a tireless agent back into a very expensive autocomplete.

🎩

The Mister Claude way

Let him run wild — inside a sealed room. The container is the boundary; the firewall is the lock. He can't reach what isn't there.

How it works

Three moving parts. Total isolation.

1

You run mrc ~/your/repo

The Node.js launcher auto-starts Colima, builds a Docker image stamped with your UID/GID, discovers your .sandboxignore files, spins up in-process clipboard & notification proxies, and boots the container with your repo bind-mounted at /workspace.

2

The firewall jams their radar

Inside the container, iptables sets a default-DROP policy and an ipset allowlist. Only a few domains resolve. IPv6 is blocked. Host services — Postgres, Redis, that staging box — are unreachable. It verifies the lock by confirming example.com is dead.

3

Claude runs with the gloves off

--dangerously-skip-permissions is perfectly safe here, because the container is the sandbox. He installs packages, runs tests, rewrites files — and VS Code on your host sees every change instantly through the bind mount.

The security boundary

He can only reach five doors. Everything else is a wall.

The allowlist is resolved to IPs at boot and pinned in an ipset. Outbound traffic to anything else gets an immediate REJECT — fast feedback, no hanging sockets.

Need the open web?

Pass --web (or set it in .mrcrc) to open outbound HTTPS for web search and fetch — while every other restriction stays in place. Add your own private registries by editing one loop in init-firewall.sh.

sessions

Now with a crew

Don't run one agent. Command a team.

Mister Claude can run a whole team of agents — an architect, engineers, a critic, an adversary — each sealed in its own container behind the same firewall, addressing each other by @mention. You steer from a web dashboard or your phone. They propose; you steer; you commit. The crew works; you stay in command.

🤝

Agent teams

Declare a roster (or pick a preset) and mrc team up. Each member boots in its own sandbox with its own write territory, so they never clobber each other's files. They collaborate by @mention; a critic reviews risky work; you commit.

orchestration
🖥️

Live dashboard

One local web app for the whole org: every team room, the @you inbox, live transcripts. Watch, steer or pause any member, suspend a project and resume it later — each member picks up its own conversation. Built in, no extra service.

built-in
📱

Telegram to your phone

Link a bot and the team's questions and updates push to your phone with stable #ids. Reply from Telegram and it routes back to the right agent. A quick BotFather + Confirm to link, then you're set.
