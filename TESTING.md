So you no longer need a manual `pkill -f room-daemon.js` after a code change — `mrc rooms restart` always comes up on the new version. If it genuinely can't free the port (no `lsof`/`ps`), it **fails loud** instead of silently serving stale code. Takes effect immediately (host CLI, no rebuild)

A short history (≤ the window) is unchanged ("N calls", cards from #1)
